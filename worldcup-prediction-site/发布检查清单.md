# 发布检查清单

## 每次更新内容

1. 将新的 Markdown 放入原始素材目录。
2. 回到项目目录运行：

```powershell
python .\scripts\import_content.py
```

3. 打开本地预览：

```powershell
python .\scripts\serve.py
```

4. 检查首页、比赛详情页、轮次页、球队页是否正常。
5. 运行构建：

```powershell
npm run build
```

如果当前机器没有安装 Node.js，可改用：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build.ps1
```

## 发布到互联网

推荐使用 GitHub Pages 自动发布：

1. 把整个仓库推送到 GitHub。
2. 打开 GitHub 仓库 `Settings -> Pages`。
3. `Source` 选择 `GitHub Actions`。
4. 等待 Actions 自动部署完成。
5. 使用 GitHub 生成的 `github.io` 链接分享。

## 上线前检查

- 首页能看到比赛预测卡片。
- 单场专题页能打开完整 Markdown 内容。
- 球队历史页能按球队聚合比赛。
- 搜索框可用。
- 背景图能加载。
- 页面底部有免责声明。
- 原始素材目录没有被修改。
