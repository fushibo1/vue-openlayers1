# 发布检查清单

## 每次更新内容

1. 将新的 Markdown 放入原始素材目录。
2. 回到项目目录运行：

```powershell
python .\scripts\import_content.py
```

3. 本地预览：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\dev.ps1
```

4. 检查首页、比赛详情页、轮次页、球队页是否正常。
5. 运行构建：

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\build.ps1
```

## 发布到互联网

推荐使用 Netlify 自动发布：

1. 打开 Netlify。
2. 选择 `Add new site -> Import an existing project`。
3. 选择 GitHub 仓库 `fushibo1/vue-openlayers1`。
4. Netlify 会读取仓库根目录的 `netlify.toml`。
5. 点击部署。

## 上线前检查

- 首页能看到比赛预测卡片。
- 单场专题页能打开完整 Markdown 内容。
- 球队历史页能按球队聚合比赛。
- 搜索框可用。
- 背景图能加载。
- 页面底部有免责声明。
- 原始素材目录没有被修改。
